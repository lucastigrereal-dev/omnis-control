## 1. Architecture Decision

```
┌──────────────────────────────────────────────────┐
│                    KRATOS                         │
│          (Mission Control Frontend)               │
│    Porta de entrada geral do ecossistema          │
│                                                   │
│   ┌─────────┐   ┌──────────┐   ┌─────────────┐  │
│   │ KRATOS  │   │  OMNIS   │   │   JARVIS     │  │
│   │ native  │   │ Cockpit  │   │  Dashboard   │  │
│   │  views  │   │ (iframe) │   │  (future)    │  │
│   └─────────┘   └──────────┘   └─────────────┘  │
└──────────────────────────────────────────────────┘
                         │
                         ▼
┌──────────────────────────────────────────────────┐
│              OMNIS Cockpit (standalone)            │
│         http://localhost:3100/cockpit/             │
│                                                   │
│   Static HTML + vanilla JS + CSS                  │
│   Zero dependencies. No framework. No build step. │
│   Served by Python http.server or open-webui.     │
│                                                   │
│   Reads: filesystem JSON/MD only (read-only)      │
│   Writes: nothing (no POST/PUT endpoints)         │
│   Network: zero external calls                    │
└──────────────────────────────────────────────────┘
```

**KRATOS ↔ OMNIS boundary:**
- KRATOS embeds OMNIS Cockpit via `<iframe>` pointing to `http://localhost:3100/cockpit/index.html`
- KRATOS never accesses OMNIS filesystem directly
- OMNIS Cockpit runs fully standalone — can be opened directly in browser
- KRATOS menu adds a "OMNIS" nav item that loads the iframe
- Authentication: if KRATOS has auth, it passes a read-only token via URL param (`?token=...`)
- No shared state between systems. No shared database. No shared JS globals.

### Route Map

| Route | File | Purpose | Data Sources |

### Navigation Structure

```
Top Bar (sticky)
├── 🏠 Dashboard
├── 📋 Missions
├── 📝 Backlog
├── ⭐ Quality
├── 📄 Templates
├── ⚡ Runs
├── 🔍 Search
└── ⚙️ System
```

### Screen: Dashboard (`index.html`)

**KPI Cards (top row):**
- Active missions: count + % completed today
- Backlog items: pending / in_progress / blocked
- Quality: avg score (A/B/C/D/F) + last scored
- Templates: total + by category
- System: disk free % + container health summary

**Second row — Recent Activity:**
- Last 5 completed missions with timestamps
- Last 5 quality reports with scores
- Last 5 backlog changes

**Third row — Alerts (if any):**
- Backlog blocked items
- Failed parallel runs
- Quality scores below threshold (D/F grades)

### Screen: Missions (`missions.html`)

- Table: ID, Status, Sector, Objective, Outputs, Exports, Report
- Filters: status (open/closed), sector dropdown, text search
- Sort: by timestamp, by status, by outputs count
- Click row → mission-detail.html

### Screen: Mission Detail (`mission-detail.html`)

- Header: mission ID, status badge, sector, timestamps
- Objective text
- File tree with download links (from outputs_manifest.json)
- Checksum verification status (from checksums.json)
- Report preview (first 300 chars of relatorio_final.md)
- Back to list link

### Screen: Backlog (`backlog.html`)

- Kanban columns: Pending | In Progress | Blocked | Completed
- Cards: title, type badge, priority indicator, tags
- Counts header: total per column
- Click card → expand detail (description, metadata, timestamps)
- Read-only (no drag-and-drop, no status change from UI)

### Screen: Quality (`quality.html`)

- Summary: avg score, grade distribution chart (bar chart CSS-only)
- Table: output_id, output_type, overall_score, grade, ready_for_use
- Click row → dimension breakdown (clarity, hook, SEO, CTA, tone, risk)
- Filter: by type (caption/carrossel/reel/dm/app), by grade, by ready status

### Screen: Templates (`templates.html`)

- Category tabs: All | Marketing | Ops | Automation
- Cards grid: template name, category badge, status badge, version
- Click card → expand: description, tags, file path
- Search by name or tag

### Screen: Runs (`runs.html`)

- Table: batch_id, status, tasks, succeeded, failed, duration, created_at
- Click row → expand: individual task results
- Summary header: total runs, success rate, avg duration

### Screen: Search (`search.html`)

- Search bar with source type filter checkboxes
- Results list: title, source type badge, relevance snippet
- Click result → navigate to source (mission detail, template, etc.)

### Screen: System (`system.html`)

- Module health table: name, namespace, status, test counts
- Container status: healthy/unhealthy counts
- Disk: free % with color indicator (>20% green, 10-20% yellow, <10% red)
- Memory sources available
- Alerts summary

### Data Loading Pattern

```
1. Page loads → fetch pre-generated JSON from /cockpit/data/*.json
2. Data is static files generated by Python backend
3. No fetch() to external URLs
4. No XHR/POST — pure GET of local JSON files
5. Fallback: if JSON missing, show "No data. Run: python -m src.reports.cockpit_generator"
```

### CSS Architecture

```
styles/
├── variables.css     — CSS custom properties (colors, spacing, fonts)
├── base.css          — Reset, typography, dark theme
├── layout.css        — Grid, nav, container, responsive
├── components.css    — Cards, tables, badges, kanban
└── print.css         — Clean print styles (no dark bg)
```

## 6. Security Boundaries

### Read-Only by Default
- Cockpit reads filesystem JSON only
- No write endpoints. No POST. No PUT. No DELETE.
- No form submissions. No state mutations from UI.
- Actions (complete mission, approve, enqueue) are CLI-only

### No External World
- Zero CDN links (no Google Fonts, no Tailwind CDN, no npm imports)
- All CSS/JS is local static files
- No analytics, no tracking, no telemetry

### KRATOS Integration Security
- KRATOS embeds via `<iframe sandbox="allow-scripts">`
- OMNIS Cockpit never accesses `window.parent` or `window.top`
- URL params only: `?token=read-only-guid`
- Token validated by KRATOS gateway before iframe loads
- OMNIS does not validate tokens — trusts the gateway

### File System Safety
- Cockpit reads from whitelisted paths only:
  - `missions/*/mission_contract.json`
  - `data/backlog.json`
  - `data/quality_scores.jsonl`
  - `data/runs/*.json`
  - `templates/template_registry.json`
- Never reads: `.env`, `secrets/`, `*.key`, `*.pem`

## 7. KRATOS Integration Point

### How KRATOS Points to OMNIS

```html
<!-- In KRATOS navigation -->
<a href="/omnis" target="omnis-frame">OMNIS Cockpit</a>
<iframe name="omnis-frame" src="http://localhost:3100/cockpit/index.html"
        sandbox="allow-scripts"
        style="width:100%;height:100vh;border:none;">
</iframe>
```

### KRATOS Menu Addition

```javascript
// KRATOS adds to its sidebar:
{
  id: "omnis",
  label: "OMNIS",
  icon: "cpu",
  action: "load-iframe",
  url: "http://localhost:3100/cockpit/index.html"
}
```

### What KRATOS Never Does
- Never reads OMNIS filesystem
- Never imports OMNIS Python modules
- Never writes to OMNIS data/

### What OMNIS Never Does
- Never references KRATOS DOM
- Never imports KRATOS modules
- Never assumes KRATOS auth context
