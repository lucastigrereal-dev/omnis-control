# OMNIS Project OS

Sistema de governança operacional do OMNIS CONTROL.
Documentação viva — mantida por Claude Code e validada pelo Lucas.

## Estrutura

| Arquivo | Função |
|---|---|
| `PROJECT_OS.md` | Identidade, arquitetura, autoridade |
| `CURRENT_STATE.md` | Estado atual do repositório |
| `ROADMAP.md` | Visão de longo prazo (verde/amarelo/vermelho) |
| `WAVE_REGISTRY.md` | Todas as waves com status |
| `ACTIVE_WORKTREES.md` | Worktrees ativos e recomendações |
| `GUARDRAILS.md` | Regras absolutas e limites |
| `RUNBOOK.md` | Protocolos operacionais |
| `SKILL_MATRIX.md` | Matriz de habilidades e permissões |
| `ERROR_PLAYBOOK.md` | Erros conhecidos e respostas |
| `DECISION_TREE.md` | Árvore de decisão por cenário |
| `RELEASE_CANDIDATE_CHECKLIST.md` | Checklist de RC |
| `WAVE_EXECUTION_MANUAL.md` | Manual detalhado de execução de wave |
| `APP_FACTORY_MANUAL.md` | Manual da App Factory (G14) |
| `CCOS_RUNTIME_MANUAL.md` | Manual do Runtime CCOS |
| `FINALIZATION_MANUAL.md` | Como fechar ciclos |
| `DAILY_OPERATION.md` | Comandos do dia-a-dia |
| `CONTEXT_HANDOFF_TEMPLATE.md` | Template de passagem de contexto |
| `WAVE_CLOSE_TEMPLATE.md` | Template de fechamento de wave |
| `SPRINT_CLOSE_TEMPLATE.md` | Template de fechamento de sprint |
| `SETUP_REPORT.md` | Relatório de setup do pack |

## Como usar

1. Abra este diretório no Claude Code
2. Siga `DAILY_OPERATION.md` para fluxo diário
3. Consulte `DECISION_TREE.md` quando tiver dúvida
4. Use os comandos em `.claude/commands/` via `/`
