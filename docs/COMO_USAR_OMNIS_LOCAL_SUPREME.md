# Como Usar o OMNIS Local Supreme
**Para:** Lucas Tigre | **Data:** 2026-05-18

## O que é
Sistema local autônomo que gera pacotes de produção de conteúdo, vídeo, design, apps e skills — sem publicar, sem conectar serviço externo, sem pedir permissão para o Google ou Instagram.

## O que já faz
1. Campanha 30 dias (legendas + roteiros + calendário + proposta)
2. Carrossel Premium (copy + briefing Canva + direção visual)
3. Pacote de Reels (10 roteiros + hooks + textos)
4. Blueprint de App (PRD + schema + API + frontend)
5. Criar Skill Python funcional
6. Daily/Weekly briefing autônomo
7. Cockpit HTML local para navegar outputs

## Como criar uma campanha
```
Crie uma missão Content Factory para o perfil @oinatalrn.
Tema: praias de Natal em junho.
Objetivo: atrair collab com pousadas.
Gere: 30 legendas SEOgram + calendário + proposta comercial.
Não publique. Não conecte Instagram.
```

## Como criar um carrossel
```
Crie uma missão Design Engine.
Tema: 7 praias secretas de Natal.
Perfil: @oinatalrn.
Tom: autoridade + convite.
Gere: estrutura 10 slides + copy + briefing Canva + legenda.
```

## Como criar pacote de Reels
```
Crie uma missão Video Engine.
Tema: família na praia, agosto.
Perfil: @afamiliatigrereal.
Gere: 10 roteiros + hooks + textos de tela + briefing de edição.
```

## Como criar blueprint de app
```
Crie uma missão App Factory.
App: CRM de Influenciadores Hotéis.
Domínio: CRM/vendas.
Usuários: Lucas + assistente comercial.
Gere: PRD + schema SQL + API + frontend spec.
```

## Como criar uma skill nova
```
Crie uma missão Capability Forge.
Skill: calculadora de CPM vs Meta Ads.
Input: seguidores, engajamento, valor cobrado.
Output: comparação CPM + recomendação de preço.
```

## Como abrir o cockpit
Abrir no browser: `cockpit/index.html`
Navegar por: missões → outputs → approvals.

## Como revisar outputs
1. Abrir `docs/OMNIS_LOCAL_SUPREME_OUTPUT_INDEX.md`
2. Localizar a missão e arquivo desejado
3. Abrir o .md diretamente
4. Decidir: usar / revisar / promover / descartar

## O que não fazer
- NÃO pedir para publicar no Instagram diretamente
- NÃO pedir para enviar e-mail ou WhatsApp
- NÃO deixar credentials em arquivos gerados
- NÃO rodar `git push` sem revisar primeiro
- NÃO deletar pasta `missions/` sem backup

## Checklist semanal
- [ ] Abrir cockpit e revisar outputs pendentes
- [ ] Escolher 1-3 outputs para usar/publicar manualmente
- [ ] Criar nova missão semanal (content + video + design)
- [ ] Revisar proposta comercial e prospectar 1-2 parceiros
- [ ] Commitar novos outputs gerados
