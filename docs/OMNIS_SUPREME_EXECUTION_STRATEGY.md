# 4. ESTRATÉGIA DE EXECUÇÃO

## Princípios

1. **Output visível primeiro** — cada feature entrega arquivo real no disco
2. **Dinheiro hoje** — prioridade máxima = conteúdo (produto) + prospecção (venda)
3. **Mock → Real progressivo** — cada módulo funciona mock primeiro, ativa quando validado
4. **Lucas vê → Lucas aprova → Automa** — nada vai autônomo sem validação visual

## Fases

### FASE 1: CREATION ENGINES (agora → 1-2 semanas)
**"O OMNIS cria conteúdo visual de verdade"**

```
Week 1: Video Engine
  ├── Moviepy + FFmpeg pipeline
  ├── Corte automático por hook detection
  ├── Legenda dinâmica (burned-in captions)
  ├── Zoom inteligente (face tracking)
  ├── Thumbnail automática
  ├── Export: Reels (9:16), Shorts (9:16), TikTok (9:16), YT (16:9)
  └── Output: .mp4 no disco → Lucas assiste e aprova

Week 2: Carousel Engine + Image Engine
  ├── Pillow + templates
  ├── Slide design automático (headline + body + CTA)
  ├── Export: sequência PNG + legenda JSON
  ├── Image Engine: arte feed/stories/thumb
  └── Output: PNGs no disco → Lucas vê e aprova
```

### FASE 2: INTELLIGENCE LAYER (semanas 2-3)
**"O OMNIS pesquisa e entende antes de criar"**

```
├── MemoryRouter → REAL (conectar Akasha pgvector + Qdrant + Obsidian)

```
