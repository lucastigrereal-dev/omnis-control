## Technical Writing OMNIS

**Padrão de documentos:**
- README: contexto, instalação, uso, arquitetura, contribuição
- RUNBOOK: problema → diagnóstico → solução → prevenção
- CHANGELOG: formato Keep a Changelog (Added/Changed/Fixed/Removed)
- SPEC: contexto → problema → solução → alternativas consideradas → riscos

**Tom e estilo:**
- Português para docs internos, inglês para código e comments
- Direto ao ponto — sem floreios
- Exemplos de código sempre que possível
- Tabelas para comparações, listas para passos sequenciais

**Estrutura de SKILL.md:** frontmatter (name/description/version/tags) seguido de
seções: ## O que é, ## Como usar, ## Exemplos.
